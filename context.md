Задача хакатона:
«AITU Hackday: Time for Industry 4.0 is now»
Цель задачи: Разработка модели систематизации и катологизации гидротехнических
сооружений Жамбылского региона
Примерная архитектура:
• Сбор данных — получение сведений по гидротехническим сооружениям
Жамбылского региона: каналы, гидропосты, шлюзы, водозаборы, насосные
станции, плотины, дамбы и другие объекты.
• Формирование цифрового каталога — создание единой базы объектов с
указанием наименования, типа, местоположения, технических параметров,
состояния и принадлежности.
• Геопривязка объектов — отображение сооружений на карте с координатами,
привязкой к районам, водным объектам и ирригационным системам.
• Систематизация данных — группировка объектов по типу, назначению,
техническому состоянию, уровню значимости и эксплуатационным
характеристикам.
• Оценка состояния объектов — анализ технического состояния сооружений на
основе паспортных данных, обследований, фотофиксации и эксплуатационной
информации.
• Классификация по категориям — распределение объектов по уровням:
исправное состояние, требует наблюдения, требует ремонта, аварийное
состояние.
• Визуализация и отчётность — веб-панель с картой, карточками объектов,
фильтрами, аналитикой и автоматическим формированием отчётов.
Основные задачи команд:
• Разработка структуры цифрового каталога гидротехнических сооружений.
• Сбор и внесение исходных данных по объектам Жамбылского региона.
• Разработка критериев систематизации и классификации объектов.
• Создание базы данных с геопривязкой и техническими характеристиками.
• Разработка модели оценки технического состояния сооружений.
• Реализация веб-интерфейса для просмотра, поиска и анализа объектов.
• Формирование отчётных форм и аналитических сводок по состоянию
гидротехнической инфраструктуры.

Основной результат проекта — веб-портал / сайт / мобильное приложение, обеспечивающее сбор, визуализацию и анализ данных из различных источников с привязкой к координатам. Система должна отображать состояние гидрологических постов, гидротехнических сооружений и связанных объектов на интерактивной карте, а также формировать аналитические показатели по их техническому состоянию.
Задачи:
1. Обеспечение возможности интеграции решения в общую систему 
2. Обеспечение возможности интеграции данных из различных источников в предложенный модул
3. Разработать алгоритм поиска гидротехнических сооруженийНа основе координат, картографических данных, спутниковых снимков или открытых источников определить возможное местоположение гидротехнических сооружений. 
4. Предложить алгоритм сравнения найденных объектов с существующей базой данныхПроверить, есть ли объект в базе, совпадают ли координаты, название, тип сооружения и его характеристики. Если объект отсутствует — пометить его как новый или требующий проверки. 
5. Предложить модель оценки периода осмотраНа основе состояния объекта, даты последнего осмотра, возраста сооружения, аварийности, важности объекта и сезонных факторов определить, как часто его нужно проверять. 
6. Предложить модель определения необходимости ремонтаСистема может присваивать объекту статус:норма / требуется осмотр / требуется ремонт / критическое состояние.
